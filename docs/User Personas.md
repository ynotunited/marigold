# 7. User Personas

The platform must support multiple user types with different goals and permissions.

---

## Persona 1 — Corporate Procurement Officer

### Profile

Responsible for sourcing branded merchandise for an organization.

### Goals

- Purchase products in bulk.
- Request quotations.
- Receive invoices.
- Track orders.
- Re-order previous purchases.

### Pain Points

- Slow quotation process.
- Poor communication.
- Limited product information.
- Difficult approval workflows.

### Success Criteria

The user should be able to:

- Find products quickly.
- Request quotations in under three minutes.
- Receive confirmation instantly.
- Easily reorder previous purchases.

# 13. Navigation Principles

Navigation must always answer three questions:

1. Where am I?

2. What can I do next?

3. How do I return?

Every page should make these obvious.

---

# 14. UX Principles

The interface should prioritize:

Recognition over recall.

Consistency.

Accessibility.

Speed.

Visual hierarchy.

Large touch targets.

Minimal clicks.

Meaningful animations.

Helpful empty states.

Clear error messages.

---

# 15. Animation Philosophy

Animations should never feel decorative.

Every motion should have purpose.

Examples include:

- Hero text reveals.
- Scroll-triggered section entrances.
- Product cards fading and lifting into view.
- Sticky header transitions.
- Button hover effects.
- Smooth cart drawer animations.
- Loading skeletons.
- Page transitions.
- Toast notifications.
- Form validation feedback.

Animation duration should generally stay between 200–500ms for responsiveness, with slightly longer hero animations where appropriate.
